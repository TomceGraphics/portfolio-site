The HCI Problem

High Cognitive Load During Medical Distress

When individuals experience symptoms, their cognitive capacity is inherently lowered due to stress. Traditional hospital directories ignore this, requiring patients to possess high medical literacy just to select the right department.

"DocMatch shifts the translation burden from the human to the machine. You type how you feel; the system handles the complex medical routing."

Complete Architecture

A Unified Provider Ecosystem

DocMatch is more than a patient frontend. It features a fully-fledged Provider Access portal, Doctor Dashboard, and Patient Record system, all synced via Supabase.

Smart Scheduling

Conflict-free date and time selection seamlessly integrated into the AI matching flow.

Provider Dashboard

Real-time queue management, allowing specialists to see who is waiting and review AI triage notes.

Patient Records

Detailed clinical overviews combining patient-reported symptoms with previous vital metrics.

Secure Access

Clean, friction-free authentication portals for medical staff and administrators.

Vanilla First

Built entirely without bulky frameworks. ES6+ JavaScript handles complex state management, routing, and UI rendering instantly.

Supabase Backend

PostgreSQL integration ensures data persists securely across the patient booking portal and the provider dashboard in real-time.

Graceful AI Degradation

If the Gemini 2.5 API hits quota limits, the system seamlessly falls back to a custom local keyword-matching algorithm.
